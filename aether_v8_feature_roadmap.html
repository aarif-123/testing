Claim conflict map
Parse every indexed paper's key claims and embed them alongside citation edges. When two papers make opposing claims about the same variable (e.g. "dropout improves generalization" vs. "dropout harms low-data regimes"), surface a conflict node in the knowledge graph view. Researchers click conflicts to see both sides side-by-side with AI-mediated synthesis. Color-code by resolution status: unresolved (amber), partially reconciled (blue), consensus reached (teal).
No academic tool does this. Elicit, Semantic Scholar, and ResearchRabbit all show agreement networks — not disagreement topology. This makes Aether the only tool for adversarial literature review.
Autonomous research agent
A background job system where a researcher submits a question with a depth setting (Quick / Standard / Deep). The agent decomposes into sub-questions, runs parallel graph queries, validates sources, synthesizes a structured report, and notifies the user via a badge on return. The Queue tab becomes the agent job manager. Reports land directly in Notebooks with citations pre-linked. Support agent chaining: "after this finishes, run a gap analysis on the output."
Perplexity does async search. No research tool does async synthesis with verification + notebook delivery. This collapses a 4-hour manual workflow to a single submission.
Hypothesis generator
After a researcher has accumulated 10+ papers in a Collection, trigger an analysis mode that maps the conceptual space of studied variables, finds under-explored intersections, and proposes 3–5 novel hypotheses with supporting literature and a suggested methodology sketch. Each hypothesis links to which gap in the graph it addresses. Researchers can accept (sends to Notebook), reject, or ask the agent to refine. Track accepted hypotheses in a dedicated "Research Log" with timestamps — building a personal intellectual history.
This is the holy grail feature for academic users and exists nowhere. It turns Aether from a retrieval tool into a creative research partner.

Shared research rooms
A "Room" is a shared Aether session with a unique URL. Participants see each other's cursors in the Drafting Canvas, can highlight chat responses and attach sticky notes, and co-manage a shared Reading Queue. One member's graph query results are visible to all. Roles: Owner, Editor, Viewer. Add a threaded comment system on any cited paper — turning Aether into a lab's shared research memory, not just one person's tool. This creates the strongest retention mechanism possible: switching costs go up dramatically when a whole team is embedded.
PDF deep lens
Drag any PDF (preprint, proprietary report, thesis) into Aether. It runs OCR + semantic chunking, extracts figures with captions as image nodes in the graph, and identifies methodology sections for the gap analysis engine. The paper is tagged as "private" (not shared to global index). Figure extraction is a killer differentiator — researchers constantly lose track of which paper contained which diagram. Store extracted figures in Collections with back-links to the source page.
Evidence strength meter
For every claim Aether surfaces, calculate an evidence score (0–100) derived from: number of papers supporting it, combined sample size, whether replications exist, recency weight, and methodology classification (RCT > observational > theoretical). Show a five-segment bar beside each answer. Researchers can click to see the breakdown. In the Citation Manager, add an "Evidence score" column to the bibliography. This lets a researcher immediately spot which claims are shaky — a critical need in fast-moving fields like ML where preprints dominate.
Reference manager sync
OAuth connection to Zotero and Mendeley APIs. On first connect, import the researcher's full library into Aether Collections with tags preserved. Any paper saved in Aether auto-exports to their connected library. This removes the "I already have a system" objection — Aether becomes additive, not a replacement. Bonus: the imported library seeds the knowledge graph with the researcher's existing domain expertise, making Aether's recommendations immediately more relevant to their specific niche from session one.

Researcher memory profile
A persistent profile that tracks: your primary research domain (inferred from queries), frequently cited authors, preferred citation format, writing vocabulary (learned from Notebook drafts), and ongoing research threads. This profile seeds every new chat session with a system prompt that contextualizes your work. After 5 sessions, Aether proactively surfaces newly published papers in your domain without being asked. The profile is visible and editable in Settings — researchers must own their data.
Voice research mode
Web Speech API for input; TTS for Aether's responses with a research-optimized voice that reads citations as inline footnotes. A focus mode disables the visual UI and renders only a minimal waveform — designed for researchers who switch between reading physical papers and querying Aether. Voice commands: "save this to collections," "add to reading queue," "open timeline." This is the only research assistant designed for the physical-digital context switch that real researchers live in.
Aether public API
REST endpoints for: graph query (returns cited answers + confidence scores), paper lookup by DOI, collection read/write, citation export. Priced per-call for institutions. This is the platform play that turns Aether from a SaaS into infrastructure — universities build custom research portals on top, journals use it for peer-review assistance, and it creates a data flywheel (more API queries → better graph coverage). Start with a 1000-call free tier to drive adoption.
Journal fit scorer
Paste a manuscript abstract. Aether embeds it, compares against a curated index of 10,000+ journals indexed by topic vectors, impact factor, acceptance rate, and average review time. Returns a ranked list with fit score, estimated acceptance probability, and a one-sentence rationale. Link directly to the journal's submission portal. This solves a painful post-research problem that no current tool addresses — researchers spend hours manually evaluating journals. Upsell opportunity: "Submit-ready check" reviews your full manuscript for scope alignment.
Methodology audit
For any paper in a researcher's library, run an automated red-flag scan: checks for p-values clustered just below 0.05 (p-hacking signal), sample sizes below field-specific thresholds, missing control groups, failure to mention pre-registration, and retraction status via Retraction Watch API. Outputs a methodology score with flagged issues. Researchers can run a batch audit on an entire Collection before submitting a systematic review. This builds deep trust — Aether is the only tool that tells you when to be skeptical of its own sources.
